Sentinel

AI-Driven Peer-to-Peer Traffic Intelligence Network

A city-scale adaptive traffic management system where every traffic signal operates as an autonomous, intelligent edge node. Nodes communicate peer-to-peer to coordinate green waves, predict congestion, and prioritise emergency vehicles — without central server dependency. The Honeycomb Cell module forms the hardware backbone: a tri-channel embedded unit handling mesh P2P, relay, and cloud uplink simultaneously.

Reinforcement Learning Cyclone DDS Zenoh MQTT Edge Computing ESP32 Smart City
Core Architecture
  • Each traffic light = autonomous edge node running local AI, sensor processing, and P2P communication
  • Three-channel Honeycomb Cell module: mesh P2P, relay/duplication, and server uplink — all simultaneous
  • Cyclone DDS for real-time inter-node coordination; MQTT for cloud telemetry; Zenoh as the bridging data fabric
  • Cloud layer is observer-only — no real-time commands. Intelligence lives at the edge
Intelligence Layer
  • Fatigue Index: live sprint counts, distance, recovery windows per player/vehicle
  • Momentum Index: possession %, territory, shot conversion tracking
  • Seasonal adaptation: holiday surges, school-term patterns, event-based load prediction
  • Emergency preemption: signed tokens clear corridor node-by-node without central coordination
Business Model
  • Municipal SaaS — R25–70K per intersection/year; hardware retrofit R2.2–4K per node
  • Projected savings: R40–90M per metro/year vs current maintenance + congestion costs
  • B2B data: anonymised traffic analytics sold to logistics, insurers, retail footfall
  • Driver app: free tier (heatmaps, fault alerts) + premium (green-wave routing, AI ETAs)

Moment Markets

Real-Time Predictive Overlay Engine for Live Sports Betting

A contextual micro-betting overlay layer embedded directly into live football streams. When a free kick, penalty, or VAR check triggers — an overlay surfaces instantly inside the stream, offering a one-tap prediction with a 6-second window. No bet slip. No navigation. Converts passive viewership into real-time wagering using existing sportsbook infrastructure. Designed as an additive engagement layer, not a new platform.

WebRTC Real-Time Odds Low Latency HLS ML Prediction In-Play Betting UX Strategy
Prediction Engine
  • Fatigue Index — minutes played, sprint frequency, distance, recovery windows
  • Pressure Index — scoreline tension, match importance, home/away differential, crowd data
  • Environmental factors — wind speed/direction, pitch wetness, temperature
  • Goalkeeper bias metrics — historical dive direction, reaction time, positioning drift
Technical Stack
  • Event detection via official live data feeds (Opta, Sportradar, Genius Sports)
  • Overlay delivered via WebRTC / LL-HLS with ≤2–3s latency requirement
  • Client-side JS SDK injection — no stream re-encode required
  • Timestamp-aligned lockout windows prevent arbitrage exploitation
Monetisation Thesis
  • Targets under-monetised passive viewers — no new user acquisition cost
  • 200K viewers × 30% interaction × 5 bets × R10 = R3M incremental turnover per match
  • Pilot-first strategy: 3-month free-credit mode for engagement validation
  • Long-term: personalised overlays, sponsored micro-markets, AI smart predictions

Background
